Classes summary

Area Models areas in the game, i.e. spots to do mischief in. Can generate episodes, i.e. decide if the Character is confronted with an Encounter or has to fight a monster.
Effect Right now, effects are just charactermodifiers with a certain duration in turns. They can be "overridden" by specialnessBehavior classes.
Encounter Handles basic encounter procedures. Can be "overridden" by specialnessBehavior classes
Item Can be "overridden" by specialnessBehavior classes
Marker Holds information about markers that appear on the GoogleMap
Quest Provides some basic methods for handling quests. Has to be "overridden" by SpecialnessBehavior classes to do anything useful.
Recipe Basic version of the Recipe class Used to check which items can be combined, and what the resulting item is
Requirement Requirement model class Used to check if a Character fulfills certain requirements
Shop Right now, shops just have a stock and a description. Can be "overridden" by specialnessBehavior classes.
Skill Resolve the skill by interfering with the battle object The following holds true for all funcitons in this class: hero and enemy are from the Skill user's point of view
